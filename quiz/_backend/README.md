# クイズ回答の記録（バックエンド設定手順）

対象読者: 採用担当（このクイズを運用する人）

クイズページ（`quiz/index.html`）の回答を、**既存のフォーム回答シート**（gid=1642761423 のタブ）の末尾に1行ずつ追記する。
採点（スコア・点数）は**シート側の既存の関数に任せる**。このスクリプトは正解を持たず、採点もしない。

`_backend/` はGitHub Pages では使わない設定用ファイル置き場。公開ページの動作には影響しない。

---

## 列マッピング（書き込む列だけ）

```
A  タイムスタンプ        ← 送信時刻（自動）
B  スコア               ← 触らない（シートの関数）
C  氏名を記入してください ← 氏名
D  点数(100)            ← 触らない（シートの関数）
E  01(5)               ← 第1問で選んだ番号(1〜12)
F  02(2)               ← 第2問
G  03(10)              ← 第3問
H  04(6)               ← 第4問
I  05(9)               ← 第5問
J  06(12)              ← 第6問
K  削除06(11)           ← 触らない
L  職種                 ← 職種（プロダクトエンジニア / bizops）
```

- 未回答（時間切れ等）の問題は空欄で記録。
- **B・D・K は書き込まない**ので、そこに入っている関数は壊れない。

## 事前準備（1回だけ）

- **L列の見出し行に「職種」と入れておく**（追記先の見出しに合わせるため）。
- B・D（スコア・点数）の関数は、追記行にも効くように **ARRAYFORMULA 型**にしておくのがおすすめ。
  - 例: 各セルにコピーする数式だと追記行は空のままになる。`=ARRAYFORMULA(...)` を見出し直下に1つ置く形なら、行が増えても自動計算される。

## 1. Apps Script を貼り付け

1. 対象スプレッドシートを開く → メニュー「拡張機能」→「Apps Script」
2. エディタの中身を全部消して、`Code.gs` の内容を貼り付け → 保存（💾）

## 2. ウェブアプリとしてデプロイ

1. 右上「デプロイ」→「新しいデプロイ」
2. 種類（⚙️）→「ウェブアプリ」
3. 設定:
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員**  ← 匿名の受験者が送信できるように
4. 「デプロイ」→ 初回は権限承認を許可
5. 表示される **ウェブアプリ URL**（`https://script.google.com/macros/s/..../exec`）をコピー

## 3. URL をクイズページに設定

`quiz/index.html` の先頭付近、`var ENDPOINT = "";` に URL を貼る:

```js
var ENDPOINT = "https://script.google.com/macros/s/..../exec";
```

→ コミット＆プッシュすれば本番送信が有効になる。

> 動作確認: ウェブアプリ URL をブラウザで直接開くと「OK: クイズ回答受信エンドポイントです。」と出ればデプロイ成功。
> 実テスト: クイズを1回通すと、フォーム回答シートの末尾に1行追加され、E〜J に選んだ番号・L に職種が入る。
